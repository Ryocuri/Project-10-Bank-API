14. ENDPOINT : GET /user/accounts
------------------------------------------------------------------------
[ ] Définir l'endpoint GET /user/accounts
[ ] Méthode : GET
[ ] Description : Récupérer tous les comptes bancaires de l'utilisateur
[ ] Sécurité : Bearer JWT token requis
[ ] Paramètres :
    - Authorization header (required)
[ ] Réponses :
    - 200 : Liste des comptes avec succès
      Schema : Array of Account objects (accountId, accountType, balance, accountNumber)
    - 401 : Non autorisé (token invalide ou manquant)
    - 500 : Erreur serveur interne
[ ] Définir le modèle Account dans definitions

------------------------------------------------------------------------
15. ENDPOINT : GET /user/accounts/{accountId}/transactions
------------------------------------------------------------------------
[ ] Définir l'endpoint GET /user/accounts/{accountId}/transactions
[ ] Méthode : GET
[ ] Description : Récupérer toutes les transactions d'un compte pour le mois en cours
[ ] Sécurité : Bearer JWT token requis
[ ] Paramètres :
    - accountId (path parameter, required) : ID du compte
    - month (query parameter, optional) : Mois (format YYYY-MM, défaut: mois en cours)
    - Authorization header (required)
[ ] Réponses :
    - 200 : Transactions récupérées avec succès
      Schema : Array of Transaction objects
    - 400 : Paramètres invalides (accountId incorrect)
    - 401 : Non autorisé
    - 404 : Compte non trouvé
    - 500 : Erreur serveur interne
[ ] Définir le modèle Transaction dans definitions :
    - transactionId (string)
    - date (string, format date-time)
    - description (string)
    - amount (number)
    - balance (number)
    - transactionType (string: debit/credit)
    - category (string)
    - notes (string, optionnel)

------------------------------------------------------------------------
[ ] Définir l'endpoint GET /user/transactions/{transactionId}
[ ] Méthode : GET
[ ] Description : Récupérer les détails complets d'une transaction spécifique
[ ] Sécurité : Bearer JWT token requis
[ ] Paramètres :

[ ] Réponses :
    - 200 : Détails de la transaction récupérés avec succès

    - 404 : Transaction non trouvée
    - 500 : Erreur serveur interne

17. ENDPOINT : PATCH /user/transactions/{transactionId}
------------------------------------------------------------------------
[ ] Définir l'endpoint PATCH /user/transactions/{transactionId}
[ ] Méthode : PATCH
[ ] Description : Modifier les informations modifiables d'une transaction (catégorie, notes)
[ ] Sécurité : Bearer JWT token requis
[ ] Paramètres :
    - transactionId (path parameter, required) : ID de la transaction
    - Authorization header (required)
    - Body (required) : TransactionUpdate object
[ ] Body schema (TransactionUpdate) :
    - category (string, optional) : Nouvelle catégorie
    - notes (string, optional) : Nouvelles notes
[ ] Réponses :
    - 200 : Transaction mise à jour avec succès
      Schema : Transaction object (mis à jour)
    - 400 : Données invalides
    - 401 : Non autorisé
    - 404 : Transaction non trouvée
    - 500 : Erreur serveur interne
